## 一、产品定位与目标用户

### 1.1 产品定位

**一句话定位**：
> 中国科技创业企业的专属补贴申请顾问——通过智能访谈厘清企业资质，精准匹配最优补贴政策，交付合规的申请文书。

**核心价值主张**：
- **省时**：将政策研究时间从数天缩短至10分钟
- **精准**：基于企业画像智能匹配适用政策，避免遗漏
- **专业**：生成符合官方要求的申请文书，提高通过率
- **省心**：全程指引，材料清单清晰，流程透明

### 1.2 目标用户画像

#### 主要用户：早期科技创业团队

| 维度 | 描述 |
|------|------|
| **企业阶段** | 种子期、初创期（成立1-3年） |
| **行业领域** | 人工智能、生物医药、电子信息、先进制造等科技领域 |
| **团队规模** | 5-30人 |
| **年营收** | 0-1000万元 |
| **资质情况** | 部分已有/正在申请科技型中小企业、高新技术企业认定 |
| **痛点** | 不了解有哪些补贴政策、不清楚自己是否符合条件、不会写申请材料、没时间研究政策 |

#### 次要用户：孵化器/加速器运营人员

| 维度 | 描述 |
|------|------|
| **角色** | 孵化器、众创空间、科技园区的运营经理 |
| **痛点** | 需要为入驻企业提供政策咨询服务，但缺乏专业知识和时间 |
| **需求** | 批量服务企业、提升服务质量、增强企业粘性 |

### 1.3 差异化优势

| 竞品类型 | 其定位 | 我们的差异化 |
|----------|--------|-------------|
| 政务AI产品（如昌慧达） | 面向全民的通用政务服务 | **聚焦创业补贴垂直领域，深度整合成都本地政策** |
| Grant AI产品（如Grantable） | 面向非营利组织的资助申请 | **面向中国科技创业企业，本土化政策解读** |
| 创业服务Agent（如钉钉AI创业助理） | 融资对接、商业规划 | **专注政策补贴，与融资服务形成互补** |
| 文档生成工具（如法大大） | 通用文书生成 | **结合政策解读+文书生成，端到端服务** |

#### 功能2：智能政策匹配（P0）

**目标**：基于企业画像匹配适用政策

**匹配逻辑**：
```
输入：企业画像（标签化）
      ↓
标签提取：
- 地区标签：[成都市, 高新区]
- 行业标签：[人工智能]
- 规模标签：[小微企业, 科技型中小企业]
- 资质标签：[高企认定中, 有知识产权]
      ↓
政策筛选：
- 地区匹配：政策适用地区包含企业注册地
- 行业匹配：政策适用行业包含企业所属行业
- 条件匹配：企业满足政策的申请条件
- 排他匹配：排除企业已享受/不符合的政策
      ↓
输出：匹配政策列表（按推荐度排序）
```

**匹配算法（MVP简化版）**：
```python
# 伪代码示例
def match_policies(enterprise):
    matched = []
    for policy in policies:
        score = 0
        # 地区匹配（权重30%）
        if enterprise.region in policy.regions:
            score += 30
        # 行业匹配（权重20%）
        if enterprise.industry in policy.industries:
            score += 20
        # 资质匹配（权重30%）
        if check_qualifications(enterprise, policy):
            score += 30
        # 规模匹配（权重20%）
        if check_scale(enterprise, policy):
            score += 20
        
        if score >= 60:  # 匹配度阈值
            matched.append({
                'policy': policy,
                'score': score,
                'estimated_amount': calculate_amount(enterprise, policy)
            })
    
    return sorted(matched, key=lambda x: x['score'], reverse=True)
```

### 3.3 核心服务设计

#### 服务1：政策匹配引擎

**功能**：基于企业画像匹配适用政策

**输入**：
```json
{
  "enterprise": {
    "name": "成都智创科技有限公司",
    "region": "高新区",
    "industry": "人工智能",
    "founding_date": "2023-06",
    "employees": 15,
    "revenue": 500,
    "qualifications": ["科技型中小企业"],
    "ip_count": 5,
    "rd_ratio": 0.15
  }
}
```

**输出**：
```json
{
  "matched_policies": [
    {
      "policy_id": "P001",
      "policy_name": "成都市高新技术企业认定奖补",
      "match_score": 95,
      "estimated_amount": 150000,
      "amount_calculation": "首次认定15万元（市本级）",
      "application_conditions": ["通过高企认定", "纳税关系在成都市"],
      "deadline": "2025-12-31",
      "priority": "高"
    }
  ]
}
```

**实现方案**：
```python
# 简化版匹配算法（MVP）
class PolicyMatchingEngine:
    def __init__(self, policy_repository):
        self.policies = policy_repository
    
    def match(self, enterprise: Enterprise) -> List[MatchedPolicy]:
        results = []
        for policy in self.policies.get_active_policies():
            score = self._calculate_match_score(enterprise, policy)
            if score >= 60:
                amount = self._estimate_amount(enterprise, policy)
                results.append(MatchedPolicy(
                    policy=policy,
                    score=score,
                    estimated_amount=amount
                ))
        return sorted(results, key=lambda x: x.score, reverse=True)
    
    def _calculate_match_score(self, enterprise, policy) -> int:
        score = 0
        # 地区匹配（30分）
        if enterprise.region in policy.regions:
            score += 30
        # 行业匹配（20分）
        if enterprise.industry in policy.industries:
            score += 20
        # 资质匹配（30分）
        if self._check_qualifications(enterprise, policy):
            score += 30
        # 规模匹配（20分）
        if self._check_scale(enterprise, policy):
            score += 20
        return score
```
